# Director Skill — STAGE 5: QC (defect→fix routing)

*The QC authority for the whole pipeline. Tool: `consistency_qc`. Owns **Gate A** (keyframe) and
**Gate B** (clip + cut). This is the consistency discipline — better than a generic reviewer skill.*

## The one rule
**Classify the root cause BEFORE choosing a tool.** The single most expensive mistake is **seed-sweeping
a still that is already wrong**. Diagnose at the **keyframe first**, animate second.
> The keyframe (SDXL) decides WHO/WHERE/WHAT-COLOR and is the root cause of most defects; the I2V (Wan)
> decides only HOW-IT-MOVES — so never seed-sweep a still that is already wrong.

## The debug loop (systematic, reproducible — see `../recipe/CONSISTENCY_QC.md`)
1. **DIAGNOSE.** Extract the keyframe (f0) + a mid + tail frame. Name the defect in **ONE** category:
   identity · wardrobe/colour · composition/count · framing/body-crop · prop-missing · palette ·
   motion-jitter · 2-figure-collapse · flat/static. (Two stacked defects → fix the still one first.)
2. **ROUTE TO THE KNOB** (turn exactly ONE):

| Diagnosis | Knob | Concrete move |
|---|---|---|
| Identity drift | face-IP scale ↑ / add face-crop + LoRA w→1.15 | canonical face crop @0.30 CU / 0.45 masked box |
| LoRA domination (multi-char) | plate→region-inpaint | gen no-LoRA plate, mask each region, inpaint with that char's LoRA serially |
| Wardrobe/colour bleed | prompt + targeted negative | reinforce the garment colour + an anti-`<background-colour>` negative |
| Composition / duplicate / missing | fresh seed + solo/2-shot prompt (+optional Depth-CN) | re-roll seed (re-key at the SAME seed keeps the duplicate) |
| Body-crop / floating bust | full-body prompt + anti-crop negative | "whole figure head to feet, both feet on the floor" |
| Prop missing | token order | trigger FIRST, load-bearing noun in first ~60 tokens, shorten ID text |
| Palette wrong | lighting clause in-prompt | name the light (warm vs cool) — never swap the style anchor |
| Motion jitter on a clean still | I2V seed (kill 2-3) | vary `--seed`; tail-drift → `--guide-scale`→4.5 + anti-morph neg |
| 2-figure collapse in motion | swap to `TWO_FIGURE_NEG` | drop duplicate/identity-change terms; "both remain"; fewer frames |
| Flat / static scenic | route OUT of gen | depth-parallax or emotion-hold, not a re-roll |

3. **RE-GEN, governor-gated, serial.** One heavy job at a time. Never overwrite a source.
4. **VERIFY AGAINST CANON.** Look at the actual pixels (don't trust the prompt). Verify against your
   per-character checklist — verify against the **canon**, NOT against a drifted frame. Log seed +
   knob + outcome to the shot manifest so the win is reproducible.

## The decision tree (route a wrong shot — see `../recipe/CONSISTENCY_QC.md`)
```
1. STILL wrong (count/figure/face/colour/framing)? → RE-KEY (Stage 1). Do NOT animate.
2. Still CLEAN but motion jitters/warps/boils?     → SEED-SWEEP the I2V (kill after 2-3).
3. Near-static LONG window unstable at length?     → SHORT-CORE + minterpolate.
4. Flat / static / quiet scenic beat?              → DEPTH-PARALLAX (zero regeneration).
5. Marginal / defective only at head-or-tail?      → TRIM / re-time.
6. Clean but feels un-animated / emotionally flat? → TIMING PASS (zero-model; after the still passes).
```

## The two gates (the hard stops — a shot does NOT advance with an open defect)
- **GATE A (keyframe, batch-level, before any animation):** identity · props-survive-lighting ·
  wardrobe colour · location geometry · palette · count · composition. Miss = re-roll the keyframe.
  → checklist in `01_keyframe.md`.
- **GATE B (clip + cut):** identity holds to the **tail**; no flicker at speed; props/palette; audio
  synced; crop safe. → checklist in `02_animate.md` + `03_assemble.md`.

## Per-character verify-checklist (the floor — match the PIXELS to YOUR canon)
Write one short line per character: the LoRA **trigger word**, the **wardrobe + colour**, and the
**load-bearing prop / identity anchor** (e.g. glasses, a scarf, a satchel). Then match the rendered pixels
to that line every shot. *(These lines are project-specific — define your own; do not ship a fixed cast.)*

## Helpers on disk
Provide your own small diagnostic helpers (a frame-grid extractor, a region-mask tool, a colour-audit
zoom) plus this routing doc (`../recipe/CONSISTENCY_QC.md`, the authority).
